01
Persiapan Awal Sistem
Dasar
TUJUAN MODUL

Mengubah Raspberry Pi dari kondisi kosong (belum bisa digunakan) menjadi sebuah komputer mini yang berfungsi penuh, dengan sistem operasi yang sudah terpasang dan siap dipakai.

ANALOGI SISTEM

Proses ini seperti merakit sebuah komputer baru dari komponen, lalu memasang sistem operasinya (misalnya Windows) untuk pertama kali. Tindakan ini memberikan 'otak' komputer (Raspberry Pi) perintah dasar untuk hidup dan berfungsi.

LANGKAH-LANGKAH UNTUK DIIKUTI
  1. Unduh Alat Bantu Raspberry Pi Imager Fungsi: Alat resmi ini adalah aplikasi kecil yang memastikan Sistem Operasi (OS) ditulis ke kartu MicroSD dengan benar. Memakai alat ini adalah langkah penting agar Raspberry Pi bisa menyala dengan sukses.
  2. Jalankan Imager dan Pilih Pengaturan Fungsi: Di sini dilakukan pemilihan model Raspberry Pi yang digunakan dan versi OS yang akan dipasang. Rekomendasi Raspberry Pi OS memberikan kecocokan dan dukungan terbaik.
  3. Lakukan Pengaturan Awal (Advanced Options) Fungsi: Ini adalah langkah yang sangat berguna untuk menghemat waktu. Dengan mengatur nama perangkat (hostname), mengaktifkan SSH untuk akses jarak jauh, dan memasukkan detail WiFi sebelum pemasangan, perangkat akan otomatis siap dipakai saat pertama kali dinyalakan.
  4. Pilih Kartu MicroSD dan Mulai Proses Penulisan Fungsi: Tentukan kartu MicroSD sebagai tujuan. Proses 'Write' akan menyalin semua file OS ke kartu. SISTEM ALERT: Proses ini akan menghapus semua data yang ada di kartu MicroSD secara permanen.
  5. Pasang Kartu dan Nyalakan Sistem Fungsi: Setelah selesai, masukkan kartu MicroSD ke Raspberry Pi. Sambungkan perangkat tambahan (seperti monitor, keyboard, mouse). Sambungkan kabel daya sebagai langkah terakhir untuk memulai proses menyalakan sistem untuk pertama kali.
VERIFIKASI KRITERIA SUKSES

Sistem berhasil menyala hingga menampilkan tampilan desktop. Koneksi ke jaringan WiFi terjadi secara otomatis. Tidak ada permintaan pengaturan tambahan yang muncul saat pertama kali menyala.

SUMBER ACUAN
  • Unduh Raspberry Pi Imager
02
Dasar Interaksi Berbasis Teks
Dasar
TUJUAN MODUL

Menguasai penggunaan perintah dasar pada aplikasi Terminal (aplikasi untuk mengetik perintah) untuk berinteraksi dengan sistem operasi secara tepat dan cepat tanpa menggunakan mouse.

ANALOGI SISTEM

Jika desktop bergambar adalah komunikasi dengan komputer melalui gambar dan klik, maka interaksi berbasis teks adalah komunikasi via bahasa ketik yang dimengerti oleh manusia dan sistem komputer. Metode ini lebih cepat dan lebih kuat untuk tugas-tugas pengaturan sistem.

LANGKAH-LANGKAH UNTUK DIIKUTI
  1. Buka Aplikasi Terminal Fungsi: Aplikasi ini adalah gerbang atau jendela percakapan teks langsung dengan inti sistem operasi. Ini adalah alat utama untuk interaksi berbasis teks.
  2. Jalankan Perintah pwd Fungsi: Perintah ini (Print Working Directory) adalah cara bertanya pada sistem, "Di folder mana saya aktif saat ini?"
  3. Jalankan Perintah ls -la Fungsi: Perintah ini meminta daftar lengkap semua item (file dan folder) di lokasi saat ini, termasuk detail seperti izin akses dan file yang tersembunyi.
  4. Jalankan Perintah mkdir nama_proyek Fungsi: Perintah ini (Make Directory) memberi perintah untuk 'membuat sebuah folder baru' dengan nama yang ditentukan.
  5. Jalankan Perintah cd nama_proyek Fungsi: Perintah ini (Change Directory) memberi perintah untuk 'pindah lokasi aktif' ke dalam folder yang baru saja dibuat.
  6. Jalankan Perintah nano catatan.txt Fungsi: Perintah ini membuka nano, sebuah aplikasi edit teks sederhana. Ini memungkinkan pembuatan atau perubahan file teks langsung dari Terminal. Simpan dan keluar dengan mengikuti petunjuk keyboard di bagian bawah layar.
VERIFIKASI KRITERIA SUKSES

Folder dan file berhasil dibuat melalui Terminal. Keberadaannya dapat diperiksa menggunakan perintah ls, dan isinya dapat dilihat dengan perintah cat catatan.txt.

03
Mengatur File & Izin Akses
Dasar
TUJUAN MODUL

Memahami perbedaan penting antara pengguna biasa dan superuser (pengguna dengan hak akses tertinggi, disebut root). Mengatur izin akses file untuk menjaga keamanan sistem.

ANALOGI SISTEM

Bayangkan sistem file adalah sebuah gedung apartemen. Pengguna biasa memiliki kunci untuk unit apartemennya sendiri. Superuser (saat menggunakan sudo) memiliki kunci master yang bisa membuka semua pintu, termasuk ruang mesin dan panel listrik. Izin akses adalah aturan tentang kunci mana yang bisa membuka pintu mana.

LANGKAH-LANGKAH UNTUK DIIKUTI
  1. Lakukan Percobaan Operasi Terlarang Fungsi: Coba buat file di dalam folder sistem yang penting, seperti /etc. Sistem akan menolak dengan pesan "Permission Denied". Ini adalah mekanisme pertahanan diri sistem untuk mencegah kerusakan file pengaturan penting.
  2. Gunakan Peningkatan Hak Akses dengan sudo Fungsi: Ulangi perintah yang sama, tetapi awali dengan sudo (Super User Do). Perintah ini secara efektif 'meminjam' kunci master untuk satu tugas spesifik setelah identitas diverifikasi melalui kata sandi.
  3. Periksa Izin dengan ls -l Fungsi: Perintah ini menampilkan detail file, termasuk kolom izin seperti -rwxr--r--. Ini adalah kode keamanan: r (baca), w (tulis), x (jalankan) untuk tiga kategori: pemilik, grup, dan lainnya.
  4. Ubah Izin dengan chmod Fungsi: Perintah chmod (Change Mode) adalah alat untuk mengubah kode keamanan tersebut. Izin dapat ditambah atau dikurangi untuk pengguna yang berbeda, misalnya membuat sebuah skrip bisa dijalankan atau membuat file menjadi hanya-bisa-dibaca.
VERIFIKASI KRITERIA SUKSES

Menerima pesan galat "Permission Denied" saat mencoba operasi terlarang. Berhasil melakukan perintah yang sama dengan sudo. Berhasil mengubah izin akses file dan memeriksa perubahannya melalui ls -l.

04
Pengaturan Jaringan Tetap
Dasar
TUJUAN MODUL

Mengganti alamat IP yang bisa berubah dengan alamat IP yang tetap. Ini sangat penting untuk perangkat yang perlu selalu ditemukan di alamat jaringan yang sama.

ANALOGI SISTEM

Alamat IP yang bisa berubah adalah seperti menyewa PO Box yang nomornya berubah setiap hari. Alamat IP yang tetap adalah seperti memiliki alamat rumah permanen. Untuk layanan yang perlu selalu ditemukan (seperti kamera keamanan), alamat permanen adalah sebuah keharusan.

LANGKAH-LANGKAH UNTUK DIIKUTI
  1. Gunakan raspi-config untuk Nama Perangkat Fungsi: Jalankan sudo raspi-config. Mengubah nama perangkat (hostname) memberikan nama yang mudah diingat di jaringan, misalnya pi-kamera, alih-alih nama bawaan yang rumit.
  2. Edit File Pengaturan Jaringan Fungsi: Buka file di /etc/dhcpcd.conf menggunakan sudo nano. File ini mengontrol bagaimana perangkat meminta alamat IP. Dengan mengeditnya, permintaan otomatis akan diganti dengan pengaturan manual.
  3. Tambahkan Blok Pengaturan Tetap Fungsi: Di akhir file, tambahkan baris yang menentukan alamat IP yang diinginkan (static ip_address) dan alamat router (static routers). Ini secara jelas memberitahu sistem untuk tidak lagi meminta alamat, melainkan memakai yang ini.
  4. Mulai Ulang Sistem Fungsi: Jalankan sudo reboot. Proses mulai ulang diperlukan agar perubahan pengaturan jaringan yang mendasar ini dapat dibaca dan diterapkan oleh sistem.
VERIFIKASI KRITERIA SUKSES

Setelah mulai ulang, perintah hostname -I akan menampilkan alamat IP tetap yang baru. Perangkat dapat di-ping dari komputer lain di jaringan yang sama menggunakan alamat IP tersebut.

05
Akses Jarak Jauh (SSH & VNC)
Menengah
TUJUAN MODUL

Mengaktifkan metode akses jarak jauh untuk memungkinkan pengoperasian tanpa monitor, keyboard, dan mouse (dikenal sebagai operasi "headless").

ANALOGI SISTEM

SSH adalah seperti melakukan panggilan telepon (hanya suara/teks) ke Raspberry Pi untuk memberinya perintah. VNC adalah seperti melakukan panggilan video di mana layar Raspberry Pi ditampilkan dan dapat dikontrol dari jarak jauh.

LANGKAH-LANGKAH UNTUK DIIKUTI
  1. Jalankan Alat Bantu raspi-config Fungsi: Aplikasi sudo raspi-config adalah panel kontrol pusat untuk mengaktifkan atau menonaktifkan fitur-fitur inti pada Raspberry Pi.
  2. Aktifkan Layanan SSH Fungsi: Arahkan ke "Interface Options" dan aktifkan SSH. Ini akan menjalankan layanan di latar belakang yang 'mendengarkan' permintaan koneksi berbasis teks yang aman.
  3. Aktifkan Layanan VNC Fungsi: Di lokasi yang sama, aktifkan VNC. Ini akan menjalankan layanan yang menyiarkan tampilan desktop bergambar melalui jaringan, memungkinkan aplikasi lain untuk terhubung dan berinteraksi dengannya.
  4. Hubungkan dari Komputer Lain Fungsi: Gunakan aplikasi penghubung SSH (seperti PuTTY di Windows) atau VNC Viewer (dari RealVNC) di komputer lain untuk terhubung ke alamat IP Raspberry Pi.
VERIFIKASI KRITERIA SUKSES

Berhasil masuk dan menjalankan perintah di Raspberry Pi melalui koneksi SSH. Berhasil melihat dan mengontrol seluruh tampilan desktop Raspberry Pi dari komputer lain melalui VNC Viewer.

06
Transfer File (WinSCP untuk Windows)
Menengah
PRASYARAT SISTEM

Modul 05 (Akses Jarak Jauh) harus telah diselesaikan. Layanan SSH harus aktif di Raspberry Pi dan alamat IP-nya diketahui.

TUJUAN MODUL

Membangun jalur transfer file yang aman dan mudah digunakan antara komputer Windows dan Raspberry Pi menggunakan aplikasi WinSCP.

ANALOGI SISTEM

WinSCP adalah seperti File Explorer dengan dua jendela berdampingan: satu menunjukkan file di komputer lokal, yang lain menunjukkan file di Raspberry Pi. Ini memungkinkan pemindahan file dengan cara seret-dan-lepas (drag-and-drop) yang familiar.

LANGKAH-LANGKAH UNTUK DIIKUTI
  1. Pasang Aplikasi WinSCP di Windows Fungsi: Unduh dan pasang WinSCP. Aplikasi ini adalah penghubung bergambar untuk transfer file aman (SCP/SFTP) yang memanfaatkan koneksi SSH.
  2. Atur Sesi Koneksi Baru Fungsi: Jalankan WinSCP. Masukkan alamat IP Raspberry Pi (Host name), nama pengguna, dan kata sandi. Pilih metode SCP.
  3. Terima Kunci Identitas Perangkat Fungsi: Pada koneksi pertama, sebuah peringatan keamanan akan muncul. Ini adalah prosedur standar untuk memeriksa identitas perangkat lain. Klik "Yes" untuk menyimpan dan mempercayai identitas dari Raspberry Pi.
  4. Lakukan Transfer File Fungsi: Setelah terhubung, tampilan dua panel akan muncul. Seret file dari panel kiri (komputer) ke kanan (Raspberry Pi) untuk mengirim, atau sebaliknya untuk menerima.
VERIFIKASI KRITERIA SUKSES

File yang diseret dari satu panel muncul di panel lainnya. File tersebut dapat diakses dan dibuka di perangkat tujuan, menandakan transfer data berhasil dan tanpa kerusakan.

SUMBER ACUAN
  • Unduh WinSCP
07
Dasar Penulisan Kode Python
Dasar
TUJUAN MODUL

Menguasai aturan dasar penulisan kode menggunakan bahasa Python. Ini adalah bekal untuk memberi perintah, mengatur logika, dan berinteraksi dengan komponen elektronik.

ANALOGI SISTEM

Belajar Python seperti belajar "tata bahasa" untuk berbicara dengan komputer. `Wadah data` (variabel) adalah kata benda, fungsi adalah kata kerja, dan struktur `if/else` adalah cara membuat kalimat sebab-akibat.

LANGKAH-LANGKAH UNTUK DIIKUTI
  1. Buka Aplikasi Thonny IDE Fungsi: Thonny adalah aplikasi bantu yang ramah pemula, menyediakan editor teks untuk menulis kode dan jendela untuk melihat hasilnya dalam satu tempat.
  2. Buat Wadah Data dan Cetak Isinya Fungsi: Buat sebuah wadah data (variabel) untuk menyimpan informasi seperti angka atau rangkaian teks. Gunakan print() untuk menampilkan isi wadah tersebut.
  3. Tulis Struktur Pengambilan Keputusan (if-elif-else) Fungsi: Struktur ini memungkinkan kode untuk menjalankan bagian yang berbeda berdasarkan kondisi yang terpenuhi, memberinya kemampuan untuk "membuat keputusan".
  4. Terapkan Perulangan (for dan while) Fungsi: Perulangan mengotomatisasi tugas yang berulang. Loop for ideal untuk mengerjakan sesuatu pada setiap item dalam sebuah daftar. Loop while terus berjalan selama suatu kondisi benar.
VERIFIKASI KRITERIA SUKSES

Kode Python berjalan tanpa menampilkan pesan galat. Hasil yang muncul di jendela "Shell" Thonny sesuai dengan logika yang telah ditulis.

SUMBER ACUAN
  • Tutorial Resmi Python 3
08
Kontrol Output GPIO (Lampu LED)
Menengah
PRASYARAT SISTEM

Pemahaman dasar Python (Modul 07) diperlukan.

TUJUAN MODUL

Memberi perintah dari kode Python untuk mengontrol komponen elektronik dari luar melalui pin GPIO (pin serbaguna), sehingga menghasilkan efek fisik yang terlihat.

ANALOGI SISTEM

Pin GPIO adalah seperti serangkaian sakelar lampu digital yang dapat dinyalakan atau dimatikan oleh kode. Menghubungkan LED adalah seperti memasang bola lampu ke salah satu sakelar tersebut.

LANGKAH-LANGKAH UNTUK DIIKUTI
  1. PROSEDUR KESELAMATAN: Matikan Aliran Listrik Fungsi: Selalu matikan dan cabut kabel daya Raspberry Pi sebelum menyambungkan atau melepaskan komponen apa pun dari pin GPIO untuk mencegah kerusakan akibat arus pendek.
  2. Rangkai Sirkuit LED Fungsi: Hubungkan pin GPIO ke resistor (untuk membatasi arus listrik agar LED tidak terbakar), lalu ke kaki panjang LED. Hubungkan kaki pendek LED ke pin Ground (GND) untuk melengkapi sirkuit.
  3. Gunakan Kumpulan Kode Bantu gpiozero Fungsi: Kumpulan kode bernama gpiozero menyediakan cara yang lebih mudah untuk berinteraksi dengan pin GPIO. Ini menyederhanakan detail teknis yang rumit menjadi perintah yang mudah dipahami (misal, led.on()).
  4. Buat Perulangan Berkedip Fungsi: Buat perulangan tak terbatas (while True) yang secara berurutan menyalakan LED, memberi jeda (sleep), mematikan LED, dan memberi jeda lagi. Ini menciptakan efek berkedip yang terus-menerus.
VERIFIKASI KRITERIA SUKSES

Lampu LED fisik pada breadboard berkedip secara teratur sesuai dengan jeda waktu yang diatur dalam kode, menunjukkan kontrol output digital berhasil.

SUMBER ACUAN
  • Panduan GPIO Zero
09
Membaca Data Sensor Analog (ADS1115)
Menengah
TUJUAN MODUL

Membaca data dari sensor yang menghasilkan sinyal analog (nilai yang bisa berubah-ubah, bukan sekadar on/off) menggunakan komponen tambahan ADC ADS1115.

ANALOGI SISTEM

Sensor analog berbicara dalam 'bahasa' yang berbeda dari Raspberry Pi. Pi hanya mengerti sinyal digital (on/off). Komponen ADC (Analog-to-Digital Converter) seperti ADS1115 berfungsi sebagai 'penerjemah' yang mengubah sinyal analog dari sensor menjadi sinyal digital yang dimengerti oleh Pi.

LANGKAH-LANGKAH UNTUK DIIKUTI
  1. Hubungkan Komponen ADS1115 Fungsi: Matikan Pi. Hubungkan pin VCC dan GND pada ADS1115 ke pin 3.3V dan GND pada Pi. Hubungkan pin SCL dan SDA pada ADS1115 ke pin SCL dan SDA yang sesuai pada Pi. Terakhir, hubungkan output dari sensor analog ke salah satu input ADC (misal, A0).
  2. Aktifkan Jalur Komunikasi I2C Fungsi: Jalankan sudo raspi-config dan aktifkan 'I2C Interface'. ADS1115 berkomunikasi menggunakan metode I2C, jadi jalur ini harus dibuka terlebih dahulu.
  3. Pasang Kumpulan Kode Bantu untuk ADS1115 Fungsi: Jalankan perintah pip install adafruit-circuitpython-ads1x15 di Terminal. Ini akan memasang kode yang sangat memudahkan pembacaan data dari ADS1115 tanpa perlu mengetahui detail teknis yang rumit.
  4. Tulis Kode untuk Membaca Nilai Fungsi: Di dalam kode Python, impor kode bantu yang baru dipasang. Buat koneksi ke ADC melalui jalur I2C, lalu baca nilai voltase dari input yang terhubung dengan sensor. Nilai voltase ini kemudian dapat diubah menjadi data yang lebih bermakna (seperti suhu atau kelembaban) sesuai manual sensor.
VERIFIKASI KRITERIA SUKSES

Kode berhasil menampilkan nilai voltase dari sensor. Nilai ini berubah secara logis ketika kondisi fisik di sekitar sensor diubah (misalnya, sensor cahaya ditutup dengan tangan).

10
Menampilkan Data di Terminal
Dasar
TUJUAN MODUL

Menampilkan data sensor yang dibaca secara terus-menerus di Terminal (aplikasi untuk mengetik perintah) dengan format yang jelas untuk keperluan pemantauan dan pemeriksaan.

ANALOGI SISTEM

Ini seperti membuat 'monitor detak jantung' sederhana untuk proyek. Alih-alih grafik yang rumit, data ditampilkan sebagai baris teks yang diperbarui secara berkala, memberikan gambaran langsung tentang 'kesehatan' atau status sistem.

LANGKAH-LANGKAH UNTUK DIIKUTI
  1. Buat Perulangan Tak Terbatas Fungsi: Gunakan loop while True dalam kode Python untuk memastikan proses pembacaan dan pencetakan data berjalan secara terus-menerus.
  2. Baca Nilai Sensor di Dalam Loop Fungsi: Di setiap putaran loop, panggil perintah yang membaca nilai terbaru dari sensor.
  3. Gunakan F-String untuk Merapikan Tampilan Fungsi: Gunakan perintah print() dengan f-string (misal, f"Suhu: {suhu:.1f} C") untuk membuat hasil yang terformat rapi. Ini menyisipkan nilai ke dalam teks dan memungkinkan pembulatan desimal.
  4. Terapkan Jeda (sleep) Fungsi: Gunakan time.sleep() di akhir setiap putaran. Ini memberikan jeda agar Terminal tidak dibanjiri data terlalu cepat dan untuk mengurangi beban kerja prosesor.
VERIFIKASI KRITERIA SUKSES

Terminal menampilkan baris data baru secara teratur sesuai jeda waktu. Data yang ditampilkan merespons perubahan kondisi sensor secara langsung.

11
Komunikasi Perangkat Keras
Lanjutan
TUJUAN MODUL

Memahami dan mengaktifkan aturan komunikasi perangkat keras (disebut protokol) I2C dan UART. Ini adalah "bahasa" yang digunakan Raspberry Pi untuk berbicara dengan modul-modul canggih dan mikrokontroler lain.

ANALOGI SISTEM

Bayangkan GPIO biasa adalah sakelar on/off. Protokol seperti I2C dan UART adalah seperti jalur telepon. Mereka memungkinkan percakapan yang lebih kompleks antara Pi dan perangkat lain, bukan hanya sinyal on/off.

LANGKAH-LANGKAH UNTUK DIIKUTI
  1. Jalankan raspi-config Fungsi: Gunakan sudo raspi-config untuk mengakses panel kontrol sistem.
  2. Aktifkan Jalur Komunikasi I2C Fungsi: Aktifkan I2C di "Interface Options". I2C adalah jalur komunikasi dua kabel yang memungkinkan banyak perangkat terhubung bersamaan, masing-masing dengan alamat unik.
  3. Aktifkan Jalur Komunikasi Serial (UART) Fungsi: Aktifkan "Serial Port" dan nonaktifkan "login shell" di atasnya. UART adalah metode komunikasi titik-ke-titik, ideal untuk koneksi dengan perangkat seperti modul GPS atau Arduino.
  4. Pasang Alat Bantu dan Lakukan Pemeriksaan Fungsi: Pasang i2c-tools. Hubungkan perangkat I2C dan jalankan i2cdetect -y 1. Perintah ini akan memindai jalur I2C dan melaporkan alamat perangkat yang terdeteksi, memastikan aktivasi berhasil.
VERIFIKASI KRITERIA SUKSES

Perintah i2cdetect berhasil menampilkan sebuah tabel dan menunjukkan alamat dari perangkat I2C yang terhubung. Ini memastikan bahwa jalur I2C aktif dan perangkat terhubung dengan benar.

12
Menerapkan Sensor I2C
Lanjutan
PRASYARAT SISTEM

Modul 11 (Komunikasi Perangkat Keras) harus diselesaikan.

TUJUAN MODUL

Membaca data secara spesifik dari sensor yang menggunakan aturan komunikasi I2C. Metode ini hemat pin karena hanya memerlukan dua pin GPIO (SDA dan SCL).

ANALOGI SISTEM

I2C bekerja seperti sistem alamat di sebuah jalan. Raspberry Pi adalah tukang pos (master), dan setiap sensor adalah rumah dengan nomor unik (alamat). Pi dapat mengirim permintaan ke alamat spesifik dan menerima data kembali darinya.

LANGKAH-LANGKAH UNTUK DIIKUTI
  1. Hubungkan Perangkat Keras Fungsi: Matikan Pi. Hubungkan pin SDA sensor ke GPIO2, SCL ke GPIO3, VCC ke 3.3V, dan GND ke Ground.
  2. Periksa Koneksi Fungsi: Nyalakan Pi dan jalankan i2cdetect -y 1. Kehadiran alamat sensor di hasil perintah memastikan koneksi fisik yang benar.
  3. Pasang Kumpulan Kode Bantu Khusus Sensor Fungsi: Seringkali, produsen sensor menyediakan kumpulan kode bantu. Kode ini menerjemahkan perintah Python yang mudah menjadi komunikasi I2C tingkat rendah yang rumit.
  4. Baca Data dalam Kode Fungsi: Impor kumpulan kode bantu dan aktifkan sensor di dalam kode. Panggil perintah dari sana untuk membaca data, misalnya sensor.temperature. Kumpulan kode bantu menangani semua detail protokol di belakang layar.
VERIFIKASI KRITERIA SUKSES

Kode berhasil menampilkan nilai sensor yang akurat (misalnya, suhu, kelembaban) dan nilai tersebut berubah secara responsif terhadap perubahan kondisi lingkungan.

13
Menghubungkan ke Layanan IoT (Blynk)
Menengah
TUJUAN MODUL

Mengirim data dari sensor yang terhubung ke Raspberry Pi ke sebuah tampilan visual di ponsel menggunakan layanan online bernama Blynk, sehingga data dapat dipantau secara langsung dari mana saja.

ANALOGI SISTEM

Bayangkan Raspberry Pi adalah reporter di lapangan yang mengumpulkan data. Blynk adalah studio berita di ponsel. Sebuah Kunci Akses Rahasia (Authentication Token) adalah 'kartu pers' reporter agar ia bisa mengirim laporannya ke studio, dan Komponen Tampilan adalah 'layar TV' di studio untuk menampilkan laporan tersebut.

LANGKAH-LANGKAH UNTUK DIIKUTI
  1. Atur Proyek di Aplikasi Blynk Fungsi: Buat akun dan proyek baru di aplikasi seluler Blynk. Proses ini akan menghasilkan sebuah Kunci Akses Rahasia (Authentication Token), yang berfungsi sebagai kata sandi unik dan rahasia untuk perangkat keras.
  2. Tentukan Jalur Data (Virtual Pins) Fungsi: Di dalam proyek, buat "Virtual Pin" (misal, V1). Ini adalah jalur data virtual yang menghubungkan perangkat keras dengan komponen tampilan di aplikasi.
  3. Desain Tampilan di Aplikasi Fungsi: Tambahkan Komponen Tampilan (seperti Gauge atau Labeled Value) ke panel kontrol aplikasi dan hubungkan komponen tersebut ke Virtual Pin yang telah dibuat. Ini adalah tampilan visual untuk data.
  4. Pasang Kumpulan Kode Bantu Blynk di Pi Fungsi: Jalankan perintah pip install blynklib. Perintah ini akan memasang kumpulan kode Python untuk Blynk, yang menangani semua logika koneksi ke layanan Blynk.
  5. Tulis Kode Pengiriman Data Fungsi: Dalam kode Python, gunakan Kunci Akses Rahasia untuk memulai koneksi. Buat perulangan yang membaca data sensor dan gunakan perintah blynk.virtual_write() untuk mengirim data ke Virtual Pin yang sesuai.
VERIFIKASI KRITERIA SUKSES

Komponen Tampilan di panel kontrol aplikasi Blynk pada ponsel menampilkan dan memperbarui nilai data sensor secara langsung sesuai dengan data yang dikirim oleh Raspberry Pi.

Cara Kerja Komunikasi MQTT
Menengah
TUJUAN MODUL

Memahami cara kerja metode komunikasi MQTT, sebuah standar industri untuk Internet of Things (IoT) yang sangat ringan dan hemat sumber daya.

ANALOGI SISTEM

MQTT bekerja seperti sistem papan pengumuman. Pengirim Pesan menempelkan pesan di papan dengan kategori tertentu (Topic). Penerima Pesan hanya melihat pesan dari kategori yang mereka minati. Sebuah Pusat Pesan adalah pengelola papan pengumuman tersebut.

PROTOKOL PEMAHAMAN
  1. Identifikasi Tiga Komponen Utama Fungsi: Pengirim Pesan (mengirim data), Penerima Pesan (menerima data), dan Pusat Pesan (sistem perantara yang mengelola pesan).
  2. Pahami Konsep "Topic" Fungsi: Topic adalah sebuah label teks (misal, rumah/kamar/suhu) yang berfungsi sebagai alamat atau kategori data. Ini memungkinkan penyaringan pesan.
  3. Lacak Alur Pesan Fungsi: Pengirim Pesan mengirim pesan ke sebuah Topic di Pusat Pesan. Pusat Pesan kemudian bertanggung jawab meneruskan pesan tersebut ke semua Penerima Pesan yang sedang "berlangganan" pada Topic yang sama.
VERIFIKASI KRITERIA SUKSES

Mampu menjelaskan peran setiap komponen dan alur data dalam skenario komunikasi MQTT. Memahami keuntungannya: pengirim dan penerima tidak perlu saling mengetahui keberadaan satu sama lain.

Menerapkan praktik keamanan dasar untuk melindungi perangkat IoT dari akses tidak sah dan ancaman umum, memperkuat pertahanan sistem.

ANALOGI SISTEM

Ini seperti mengamankan sebuah rumah. Mengubah kata sandi adalah mengganti kunci pintu. Mengaktifkan dinding api digital adalah menutup semua jendela kecuali beberapa yang penting. Mengamankan SSH dan MQTT adalah memasang kunci tambahan pada pintu-pintu tersebut.

LANGKAH-LANGKAH UNTUK DIIKUTI
  1. Ubah Kata Sandi Bawaan Fungsi: Gunakan passwd untuk mengubah kata sandi pengguna. Ini adalah langkah pertahanan pertama dan paling penting.
  2. Aktifkan Dinding Api Digital (ufw) Fungsi: Pasang ufw. Gunakan sudo ufw allow hanya untuk pintu data bernomor yang benar-benar dibutuhkan (misal, 22 untuk SSH, 1883 untuk MQTT). Kemudian aktifkan dengan sudo ufw enable.
  3. Amankan Akses SSH Fungsi: Edit file /etc/ssh/sshd_config. Ubah PermitRootLogin menjadi no untuk mencegah login jarak jauh menggunakan akun dengan kekuatan tertinggi.
  4. Amankan Akses MQTT Fungsi: Atur pusat pesan Mosquitto untuk memerlukan nama pengguna dan kata sandi. Ini mencegah koneksi tak dikenal, pengiriman data palsu, atau pengintipan data.
VERIFIKASI KRITERIA SUKSES

Upaya masuk sebagai root melalui SSH ditolak. Kode pengirim MQTT tidak dapat terhubung ke pusat pesan tanpa memberikan nama pengguna dan kata sandi yang valid.
