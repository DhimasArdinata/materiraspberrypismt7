01
Inisialisasi Sistem Primer
Dasar
OBJEKTIF SISTEM

Mengaktifkan unit Raspberry Pi dari kondisi non-operasional menjadi sistem dengan OS yang berfungsi penuh dan siap digunakan.

PROTOKOL EKSEKUSI
  1. Unduh perangkat lunak resmi `Raspberry Pi Imager` ke komputer utama Anda.
  2. Jalankan Imager, lalu pilih model Raspberry Pi dan Sistem Operasi (disarankan "Raspberry Pi OS").
  3. Klik ikon gerigi untuk melakukan pra-konfigurasi: atur hostname, aktifkan SSH, buat username dan password, serta masukkan detail jaringan WiFi Anda.
  4. Pilih kartu MicroSD sebagai target penyimpanan, lalu mulai proses penulisan (write). Proses ini akan menghapus semua data di kartu.
  5. Setelah selesai, masukkan kartu MicroSD ke Raspberry Pi. Hubungkan monitor, keyboard, mouse, dan terakhir, hubungkan catu daya.
VERIFIKASI KRITERIA SUKSES

Sistem berhasil boot hingga menampilkan desktop grafis tanpa memerlukan konfigurasi tambahan, dan secara otomatis terhubung ke WiFi.

02
Dasar Antarmuka Baris Perintah (CLI)
Dasar
OBJEKTIF SISTEM

Mengeksekusi perintah dasar di terminal untuk interaksi non-grafis yang efisien dengan sistem operasi.

PROTOKOL EKSEKUSI
  1. Buka aplikasi Terminal dari taskbar.
  2. Gunakan perintah `pwd` untuk menampilkan direktori (folder) Anda saat ini.
  3. Gunakan perintah `ls -la` untuk melihat daftar lengkap file dan folder di lokasi Anda.
  4. Buat direktori baru untuk proyek menggunakan perintah `mkdir`.
  5. Pindah ke dalam direktori baru tersebut menggunakan perintah `cd`.
  6. Gunakan editor teks `nano` untuk membuat dan menyunting sebuah file teks baru. Simpan perubahan dengan mengikuti instruksi di layar (Ctrl+X, Y, Enter).
VERIFIKASI KRITERIA SUKSES

Berhasil membuat struktur direktori dan file teks melalui terminal, yang perubahannya dapat diverifikasi baik melalui perintah `ls` maupun file manager grafis.

03
Manajemen File & Hak Akses
Dasar
OBJEKTIF SISTEM

Memahami perbedaan kritis antara pengguna biasa dan superuser (root), serta mengelola hak akses file untuk menjaga keamanan.

PROTOKOL EKSEKUSI
  1. Coba buat file di dalam direktori sistem seperti `/etc` menggunakan `nano`. Amati bahwa sistem akan menolak untuk menyimpan.
  2. Ulangi perintah yang sama, namun kali ini awali dengan `sudo`. Sistem akan meminta password Anda dan mengizinkan penyimpanan file.
  3. Lihat detail hak akses sebuah file menggunakan perintah `ls -l`. Perhatikan notasi `rwx` (read, write, execute).
  4. Gunakan perintah `chmod` untuk mengubah hak akses file tersebut. Misalnya, menambahkan atau menghapus izin eksekusi.
VERIFIKASI KRITERIA SUKSES

Mendapatkan galat "Permission Denied" saat melakukan operasi terlarang. Berhasil mengubah hak akses sebuah file dan memverifikasi perubahannya melalui `ls -l`.

04
Konfigurasi Jaringan Statis
Dasar
OBJEKTIF SISTEM

Mengonfigurasi parameter jaringan (Hostname & IP Statis) untuk memastikan konektivitas yang stabil dan dapat diandalkan.

PROTOKOL EKSEKUSI
  1. Buka utilitas `raspi-config` dari terminal dengan hak akses `sudo`.
  2. Navigasi ke menu Network Options untuk mengubah Hostname perangkat menjadi nama yang unik dan mudah diingat.
  3. Untuk IP Statis, edit file konfigurasi di `/etc/dhcpcd.conf` menggunakan `nano` dengan hak akses `sudo`.
  4. Tambahkan blok konfigurasi di akhir file untuk antarmuka Anda (misal: `interface wlan0`), lalu tentukan alamat `static_ip_address` yang diinginkan.
  5. Simpan file dan restart Raspberry Pi agar perubahan diterapkan.
VERIFIKASI KRITERIA SUKSES

Setelah reboot, gunakan perintah `hostname -I` untuk memastikan IP baru telah diterapkan. Perangkat dapat di-ping dari komputer lain menggunakan IP statis tersebut.

05
Akses Jarak Jauh (SSH & VNC)
Menengah
OBJEKTIF SISTEM

Membangun koneksi jarak jauh ke Raspberry Pi untuk operasi tanpa monitor (headless), baik melalui terminal (SSH) maupun desktop (VNC).

PROTOKOL EKSEKUSI
  1. Jalankan utilitas `raspi-config` dengan `sudo`.
  2. Navigasi ke menu "Interface Options".
  3. Aktifkan (Enable) layanan SSH.
  4. Aktifkan (Enable) layanan VNC Server.
  5. Dari komputer lain di jaringan yang sama, gunakan klien SSH (seperti PuTTY atau Terminal) untuk terhubung ke alamat IP Raspberry Pi.
  6. Unduh dan gunakan `RealVNC Viewer` di komputer lain untuk terhubung ke alamat IP yang sama untuk akses desktop grafis.
VERIFIKASI KRITERIA SUKSES

Berhasil masuk dan menjalankan perintah di terminal Raspberry Pi melalui SSH. Berhasil menampilkan dan mengontrol seluruh desktop Raspberry Pi melalui VNC.

06
Transfer File via WinSCP (Windows)
Menengah
OBJEKTIF SISTEM

Mentransfer file antara komputer Windows dan Raspberry Pi secara aman dan efisien menggunakan antarmuka grafis WinSCP.

PROTOKOL EKSEKUSI
  1. Pastikan SSH sudah aktif di Raspberry Pi (dari Modul 05).
  2. Di komputer Windows Anda, unduh dan instal aplikasi `WinSCP`.
  3. Jalankan WinSCP. Akan muncul jendela "Login".
  4. Pada jendela Login, pilih "File protocol" menjadi SCP.
  5. Masukkan alamat IP Raspberry Pi Anda di kolom "Host name", nama pengguna di "User name", dan kata sandi di "Password".
  6. Klik "Login". Anda mungkin akan melihat peringatan keamanan saat pertama kali terhubung, klik "Yes" untuk mempercayai host.
  7. Setelah terhubung, Anda akan melihat tampilan dua panel: panel kiri adalah file di komputer Anda, dan panel kanan adalah file di Raspberry Pi.
  8. Untuk mentransfer file, cukup seret (drag and drop) file dari satu panel ke panel lainnya.
VERIFIKASI KRITERIA SUKSES

File yang diseret dari panel kiri (komputer) muncul di panel kanan (Raspberry Pi), dan sebaliknya. File dapat dibuka di perangkat tujuan untuk memastikan transfer berhasil tanpa korupsi.

08
Kontrol Output GPIO (LED Blink)
Menengah
OBJEKTIF SISTEM

Mengontrol komponen elektronik eksternal (output) melalui pin General Purpose Input/Output untuk menghasilkan efek visual sederhana seperti LED berkedip.

PROTOKOL EKSEKUSI
  1. PENTING: Matikan dan cabut daya Raspberry Pi sebelum menyambungkan kabel apapun ke pin GPIO.
  2. Pada breadboard, rangkai sirkuit LED: Hubungkan pin GPIO (misal, GPIO17) ke resistor (misal, 330 Ohm), lalu ke kaki panjang (anoda) LED. Hubungkan kaki pendek (katoda) LED ke pin Ground (GND).
  3. Nyalakan kembali Raspberry Pi dan buka Thonny IDE.
  4. Dalam skrip Python, impor kelas `LED` dari pustaka `gpiozero` dan fungsi `sleep` dari pustaka `time`.
  5. Buat objek untuk merepresentasikan LED, dengan menyebutkan nomor pin GPIO yang digunakan.
  6. Buat perulangan tak terbatas (`while True`) yang di dalamnya secara berurutan: menyalakan LED (`led.on()`), memberi jeda (`sleep(1)`), mematikan LED (`led.off()`), dan memberi jeda lagi (`sleep(1)`).
VERIFIKASI KRITERIA SUKSES

LED berkedip secara teratur dengan interval satu detik menyala dan satu detik mati, menunjukkan kontrol output digital berbasis waktu yang berhasil.

09
Interfacing Sensor (Digital & Analog)
Menengah
OBJEKTIF SISTEM

Membaca data dari dunia nyata menggunakan sensor, baik yang mengeluarkan sinyal digital maupun analog, dan mengubahnya menjadi informasi yang dapat diproses.

PROTOKOL EKSEKUSI
  1. Sensor Digital (Ultrasonik HC-SR04): Hubungkan pin VCC ke 5V, GND ke GND, Trig ke satu pin GPIO, dan Echo ke pin GPIO lain. Dalam Python, gunakan pustaka `gpiozero` kelas `DistanceSensor`.
  2. Sensor Analog (Suhu LM35): Karena Raspberry Pi tidak memiliki input analog, perlukan ADC (Analog-to-Digital Converter) seperti MCP3008.
  3. Hubungkan ADC ke Raspberry Pi menggunakan koneksi SPI (pin MOSI, MISO, SCLK, CE0). Hubungkan output sensor LM35 ke salah satu channel input ADC.
  4. Aktifkan antarmuka SPI melalui `raspi-config`.
  5. Dalam Python, gunakan pustaka `gpiozero` kelas `MCP3008` untuk membaca nilai mentah dari ADC, lalu konversikan ke tegangan dan suhu menggunakan rumus yang sesuai.
VERIFIKASI KRITERIA SUKSES

Program berhasil mencetak nilai jarak dalam sentimeter dan suhu dalam derajat Celsius yang berubah secara logis ketika kondisi fisik di sekitar sensor diubah.

Visualisasi Data Terminal
Dasar
OBJEKTIF SISTEM

Menampilkan data sensor yang dibaca secara terus-menerus di terminal dengan format yang jelas dan mudah dibaca untuk keperluan pemantauan dan debugging.

PROTOKOL EKSEKUSI
  1. Buat skrip Python yang berisi perulangan tak terbatas (`while True`).

Terminal menampilkan baris data baru secara teratur (sesuai jeda `sleep`), dan data tersebut merespons perubahan pada sensor secara real-time.

11
Protokol Komunikasi Serial
Lanjutan
OBJEKTIF SISTEM

Memahami dan mengaktifkan protokol komunikasi serial UART dan I2C, yang memungkinkan Raspberry Pi berkomunikasi dengan berbagai macam modul dan mikrokontroler lain.

PROTOKOL EKSEKUSI
  1. Jalankan `sudo raspi-config` dan navigasi ke "Interface Options".
  2. Pilih "I2C" dan aktifkan (Enable).
  3. Pilih "Serial Port". Jawab "No" untuk login shell, dan "Yes" untuk mengaktifkan hardware serial port (UART).

  5. Instal perkakas bantu dengan `sudo apt install i2c-tools`.
  6. Untuk verifikasi I2C, hubungkan perangkat I2C dan jalankan perintah `i2cdetect -y 1`.
VERIFIKASI KRITERIA SUKSES

Perintah `i2cdetect` berhasil menampilkan sebuah tabel dan menunjukkan alamat heksadesimal dari perangkat I2C yang terhubung (misal, `40` atau `77`).

12
Implementasi Sensor I2C
Lanjutan
OBJEKTIF SISTEM

Membaca data secara spesifik dari sensor yang menggunakan protokol komunikasi I2C, sebuah metode yang efisien dalam penggunaan pin GPIO.

PROTOKOL EKSEKUSI
  1. Matikan Raspberry Pi. Hubungkan pin SDA sensor ke GPIO2, SCL ke GPIO3, VCC ke 3.3V, dan GND ke Ground.
  2. Nyalakan Raspberry Pi dan verifikasi koneksi dengan `i2cdetect -y 1` untuk melihat alamat sensor.
  3. Instal pustaka Python yang diperlukan untuk berinteraksi dengan bus I2C (misal, `pip install smbus2`) atau pustaka spesifik sensor (misal, dari Adafruit).
  4. Dalam skrip Python, impor pustaka tersebut. Inisialisasi bus I2C.
  5. Kirim perintah ke alamat sensor untuk memulai pengukuran dan kemudian baca data yang dikembalikan oleh sensor, sesuai dengan datasheet perangkat.
VERIFIKASI KRITERIA SUKSES

Skrip berhasil mencetak nilai sensor (misalnya, intensitas cahaya dalam Lux) yang akurat dan berubah secara responsif terhadap perubahan kondisi lingkungan.

Integrasi Platform IoT
Menengah
OBJEKTIF SISTEM

Mengirim data dari sensor yang terhubung ke Raspberry Pi ke dasbor online agar dapat dipantau dari mana saja melalui internet.

PROTOKOL EKSEKUSI
  1. Buat akun gratis di platform IoT seperti Thingspeak.

VERIFIKASI KRITERIA SUKSES

Data sensor muncul di dasbor online Thingspeak Anda sebagai grafik atau gauge yang diperbarui secara berkala sesuai dengan data yang dikirim.

14
Arsitektur MQTT
Menengah
OBJEKTIF SISTEM

Memahami arsitektur publish-subscribe dari protokol MQTT, protokol standar industri untuk komunikasi IoT yang ringan dan efisien.

PROTOKOL EKSEKUSI
  1. Identifikasi tiga komponen utama: Publisher (pengirim data, misal: sensor), Subscriber (penerima data, misal: aplikasi dasbor), dan Broker (server perantara).
  2. Pahami konsep "Topic", yaitu sebuah string teks yang berfungsi sebagai alamat atau kanal data (misal, `rumah/kamar_tidur/suhu`).
  3. Lacak alur pesan: Publisher tidak mengirim data langsung ke Subscriber. Publisher mengirim pesan ke sebuah Topic di Broker.
  4. Broker kemudian bertanggung jawab untuk meneruskan pesan tersebut ke semua Subscriber yang sedang "berlangganan" ke Topic yang sama.
VERIFIKASI KRITERIA SUKSES

Mampu menjelaskan peran setiap komponen dan alur data dalam skenario komunikasi MQTT yang diberikan, serta keuntungan dari arsitektur ini (kopling longgar).

15
Instalasi Broker MQTT Lokal
Lanjutan
OBJEKTIF SISTEM

Menginstal dan menjalankan server perantara (broker) MQTT di Raspberry Pi itu sendiri, menciptakan jaringan IoT lokal yang independen dari internet.

PROTOKOL EKSEKUSI
  1. Buka terminal dan instal broker Mosquitto beserta kliennya menggunakan `sudo apt install mosquitto mosquitto-clients`.
  2. Periksa status layanan untuk memastikan broker berjalan secara otomatis dengan `sudo systemctl status mosquitto`.
  3. Buka dua jendela terminal secara bersamaan.
  4. Di terminal pertama, jalankan perintah `mosquitto_sub` untuk berlangganan (subscribe) ke sebuah topik, misalnya: `mosquitto_sub -t "test/topic"`.
  5. Di terminal kedua, jalankan perintah `mosquitto_pub` untuk menerbitkan (publish) pesan ke topik yang sama, misalnya: `mosquitto_pub -t "test/topic" -m "Halo Dunia"`.
VERIFIKASI KRITERIA SUKSES

Teks "Halo Dunia" yang dikirim dari terminal kedua langsung muncul di terminal pertama, menandakan broker berhasil meneruskan pesan.

16
Klien MQTT dengan Python
Lanjutan
OBJEKTIF SISTEM

Mengintegrasikan fungsionalitas MQTT ke dalam skrip Python, memungkinkan program untuk mengirim (publish) dan menerima (subscribe) data secara terprogram.

PROTOKOL EKSEKUSI
  1. Instal pustaka Paho-MQTT untuk Python menggunakan `pip install paho-mqtt`.
  2. Buat skrip *publisher*: impor pustaka, buat objek klien, hubungkan ke alamat broker (`localhost`), lalu gunakan metode `client.publish()` di dalam sebuah perulangan untuk mengirim data.
  3. Buat skrip *subscriber* yang terpisah: impor pustaka, buat objek klien, dan definisikan fungsi callback (misal, `on_message`) yang akan dieksekusi setiap kali pesan diterima.
  4. Di skrip subscriber, hubungkan ke broker, berlangganan ke topik yang relevan dengan `client.subscribe()`, dan jalankan `client.loop_forever()` untuk mulai mendengarkan pesan.
VERIFIKASI KRITERIA SUKSES

Data yang dikirim oleh skrip publisher (misal, nilai sensor acak) muncul di konsol dari skrip subscriber yang sedang berjalan secara real-time.

17
Instalasi Database Server
Lanjutan
OBJEKTIF SISTEM

Menginstal dan mengamankan server database relasional lokal (seperti MariaDB) untuk penyimpanan data historis yang terstruktur dari sensor.

PROTOKOL EKSEKUSI
  1. Instal paket server MariaDB menggunakan `sudo apt install mariadb-server`.
  2. Jalankan skrip keamanan pasca-instalasi dengan `sudo mysql_secure_installation`. Ikuti petunjuk di layar untuk mengatur kata sandi root, menghapus pengguna anonim, dan menonaktifkan login root jarak jauh.
  3. Masuk ke antarmuka baris perintah database dengan `sudo mysql -u root -p`.
  4. Gunakan perintah SQL `CREATE DATABASE` untuk membuat database baru untuk proyek Anda.
  5. Gunakan `USE` untuk memilih database tersebut, lalu `CREATE TABLE` untuk mendefinisikan struktur tabel yang akan menyimpan data sensor (misal, kolom untuk id, timestamp, dan nilai suhu).
VERIFIKASI KRITERIA SUKSES

Berhasil login ke server database, dan perintah `SHOW TABLES;` menampilkan tabel yang baru saja Anda buat dengan struktur kolom yang benar.

18
Integrasi Python & Database
Lanjutan
OBJEKTIF SISTEM

Menyimpan data sensor yang diterima oleh skrip Python secara persisten ke dalam tabel di database MariaDB.

PROTOKOL EKSEKUSI
  1. Instal konektor Python untuk MySQL/MariaDB menggunakan `pip install mysql-connector-python`.
  2. Dalam skrip Python (misalnya, skrip subscriber MQTT), impor pustaka konektor.
  3. Tulis kode untuk membuat objek koneksi ke database, dengan menyediakan host (`localhost`), user, password, dan nama database.
  4. Buat objek "kursor" dari koneksi tersebut. Kursor adalah objek yang digunakan untuk mengeksekusi perintah SQL.
  5. Setiap kali data sensor diterima, buat string perintah SQL `INSERT` yang menyertakan data tersebut.
  6. Gunakan `cursor.execute()` untuk menjalankan perintah SQL, lalu panggil `connection.commit()` untuk menyimpan perubahan secara permanen ke database.
VERIFIKASI KRITERIA SUKSES

Setelah skrip berjalan untuk beberapa saat, jalankan kueri `SELECT * FROM nama_tabel;` di klien `mysql`. Tabel akan berisi baris-baris data sensor yang dikirim oleh skrip Python.

19
Topologi Jaringan IoT
Lanjutan
OBJEKTIF SISTEM

Merancang dan mengimplementasikan jaringan multi-perangkat yang lebih kompleks, seperti topologi bintang, di mana beberapa node sensor melaporkan data ke gateway pusat.

PROTOKOL EKSEKUSI
  1. Tentukan satu Raspberry Pi sebagai "Gateway". Perangkat ini akan menjalankan broker MQTT (Mosquitto). Pastikan Gateway memiliki alamat IP statis.
  2. Siapkan satu atau lebih perangkat lain (bisa Raspberry Pi, ESP32, dll.) sebagai "Node".
  3. Pastikan semua perangkat (Gateway dan Node) terhubung ke jaringan WiFi yang sama.
  4. Pada setiap Node, jalankan skrip publisher MQTT. Namun, kali ini, alamat broker yang dituju bukan lagi `localhost`, melainkan alamat IP statis dari Gateway.
  5. Di Gateway, Anda dapat menjalankan skrip subscriber yang berlangganan ke topik dari semua node untuk memantau lalu lintas data.
VERIFIKASI KRITERIA SUKSES

Log di broker Gateway atau skrip subscriber di Gateway menunjukkan pesan masuk dari beberapa alamat IP Node yang berbeda, membuktikan komunikasi multi-perangkat berhasil.

20
Dasar Keamanan IoT
Lanjutan
OBJEKTIF SISTEM

Menerapkan praktik keamanan dasar untuk melindungi perangkat IoT dari akses tidak sah dan ancaman umum.

PROTOKOL EKSEKUSI
  1. Ubah Kata Sandi Default: Segera gunakan perintah `passwd` di terminal untuk mengubah kata sandi default pengguna `pi`.
  2. Aktifkan Firewall: Instal `ufw` (`sudo apt install ufw`). Gunakan `sudo ufw allow` hanya untuk port yang benar-benar Anda butuhkan (misal, 22 untuk SSH, 1883 untuk MQTT). Kemudian aktifkan dengan `sudo ufw enable`.
  3. Amankan SSH: Edit file `/etc/ssh/sshd_config` dengan `sudo nano`. Ubah baris `PermitRootLogin` menjadi `no`. Simpan dan restart layanan SSH.
  4. Amankan MQTT: Konfigurasikan broker Mosquitto untuk memerlukan username dan password. Ini melibatkan pembuatan file password dan mengedit file konfigurasi Mosquitto untuk menunjuk ke file tersebut.
VERIFIKASI KRITERIA SUKSES

Upaya login sebagai `root` melalui SSH ditolak. Upaya koneksi ke port yang tidak diizinkan oleh firewall gagal. Klien MQTT tidak dapat terhubung ke broker tanpa memberikan username dan password yang valid.

21
Integrasi Web Framework Flask
Lanjutan
OBJEKTIF SISTEM

Mengintegrasikan data IoT real-time dari MQTT ke dalam backend aplikasi web menggunakan web framework Python yang ringan, Flask.

PROTOKOL EKSEKUSI
  1. Instal Flask menggunakan `pip install Flask`.
  2. Buat skrip Python utama untuk aplikasi web Anda. Impor Flask dan juga `paho-mqtt` serta pustaka `threading`.
  3. Buat sebuah variabel global untuk menyimpan data sensor terakhir yang diterima.
  4. Buat sebuah fungsi yang akan menjalankan klien MQTT dalam loop tak terbatas. Fungsi ini, ketika menerima pesan MQTT, akan memperbarui variabel global tadi.
  5. Jalankan fungsi MQTT tersebut dalam sebuah "background thread" agar tidak memblokir server web.
  6. Buat sebuah route Flask (`@app.route('/')`) yang, ketika diakses, akan membaca nilai dari variabel global dan menampilkannya ke pengguna (misal, sebagai teks sederhana atau JSON).
VERIFIKASI KRITERIA SUKSES

Setelah menjalankan skrip Flask dan mengirim data ke topik MQTT yang relevan, mengakses alamat `http://localhost:5000` di browser Raspberry Pi akan menampilkan data sensor terbaru.

22
Deployment Dasbor Web Lokal
Lanjutan
OBJEKTIF SISTEM

Membuat dan men-deploy dasbor web fungsional di Raspberry Pi yang dapat diakses oleh perangkat apa pun di jaringan lokal yang sama.

PROTOKOL EKSEKUSI
  1. Buat struktur direktori proyek: sebuah folder utama, dan di dalamnya sebuah subfolder bernama `templates`.
  2. Di dalam folder `templates`, buat file HTML (`index.html`) yang akan menjadi tampilan dasbor. Gunakan placeholder (misal, `

    Suhu: {{ data_sensor }} °C

    `) di mana Anda ingin data ditampilkan.
  3. Dalam skrip Python Flask, pada fungsi route utama, ubah `return` menjadi `return render_template('index.html', data_sensor=variabel_global_suhu)`. Pastikan `render_template` diimpor dari Flask.
  4. Pada baris terakhir skrip, saat menjalankan aplikasi Flask (`app.run()`), tambahkan argumen `host='0.0.0.0'` agar server dapat diakses dari alamat IP, bukan hanya `localhost`.
  5. Jalankan skrip, lalu buka browser di ponsel atau laptop yang terhubung ke WiFi yang sama dan kunjungi alamat `http://:5000`.
